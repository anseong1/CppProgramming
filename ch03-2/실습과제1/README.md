#실습과제

## 1. C++에서 생성자를 통하여 멤버변수의 초기화를 하는 이유를 설명하라. 
생성자를 통한 멤버 변수 초기화는 객체 생성 시 변수들을 자동으로 올바른 값으로 설정하여 쓰레기값을 방지하고 프로그램의 안정성을 높이기 위함이다. 또한 const나 참조형 변수는 생성자로만 초기화할 수 있기 때문이다.

---

## 2. 생성자 작성시 생성자 몸체에서 멤버를 초기화하는 것 보다 멤버 초기화 리스트에서 초기화하는 이유를 설명하라.
생성자 몸체에서 초기화할 경우 먼저 기본 생성이 이루어진 후 다시 값이 대입되지만, 초기화 리스트를 사용하면 이러한 중복 과정 없이 바로 초기화가 이루어져 불필요한 대입 과정을 줄일 수 있다. 그리고 `const`와 참조형 변수는 초기화 리스트로만 초기화할 수 있어 더 안전하고 효율적이다.

---

## 3. 위임, 타겟 생성자로 나누어 작성하는 이유를 설명하라.
생성자 간의 중복 코드를 줄이고 유지보수를 쉽게 하기 위함이다. 공통적으로 수행해야 하는 초기화 작업을 하나의 타겟 생성자에 모아두고, 다른 생성자들은 이를 호출(위임)함으로써 코드의 일관성과 가독성을 높일 수 있기 때문이다.

---

## 4. 함수 몸체(body)에서 생성자를 직접 호출하면 어떻게 되는지 조사해보라.
함수 몸체에서 생성자를 직접 호출하면 기존 객체를 초기화하는 것이 아니라 임시 객체가 생성되었다가 바로 소멸된다. 즉, 현재 객체에는 아무런 영향도 주지 않고 생성자를 일반 함수처럼 호출해도 멤버 변수 값이 바뀌지 않는다. 따라서 생성자를 함수처럼 몸체에서 호출하면 오류가 발생하진 않지만, 단지 현재 객체에는 아무런 변화가 없다.
