# 실습과제

## 1. 라이브러리는 헤더파일과 목적파일로 구성되어 있다. 헤더파일과 목적파일에 들어 있는 내용이 무엇인지 가가 설명 하시오.

헤더파일(.h)은 함수의 선언, 매크로 정의, 상수, 자료형 정의 등이 들어 있어 프로그램에서 해당 라이브러리의 기능을 사용할 수 있도록 알려주는 역할을 하고
목적파일(.obj)은 실제로 실행될 기계어가 들어 있는 파일로, 컴파일된 결과가 저장되어 있다.

---

## 2. 함수의 선언, 정의, 호출은 각각 어디에 존재하는가?

함수의 선언은 헤더파일에 존재하고 정의는 목적파일에 존재하며 호출은 사용자가 작성한 소스파일(.cpp)에 존재한다. 

---

## 3. <iostream> 헤더파일에 cin, cout은 extern 변수로 선언되어 있다. extern 선언과 변수선언의 차이를 설명하라. 그리고 그럼 cin, cout의 변수선언은 어디에 있는가?

extern 선언은 변수가 다른 파일에 정의되어 있음을 알리고 사용할 수 있게 하는 선언이고, 변수 선언은 실제로 메모리를 할당하여 변수를 만드는 것이다.
그리고 `<iosteam>` 헤더파일에서 cin, cout은 extern으로 선언만 되어 있고, 실제 변수는 라이브러리 목적파일 안에 정의되어 있다.

---

## 4. 라이브러리 함수나 객체를 사용하기 전에 헤더파일을 include하는 이유를 설명하라.

함수나 객체의 선언을 미리 컴파일러에게 알려주어 프로그램에서 올바르게 사용할 수 있도록 해주기 때문이다.
